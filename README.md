# SPI Platform

Платформа для управления событиями, организациями и регистрациями. Проект разработан на Python с использованием Django и Django Ninja.

## О проекте

SPI Platform предоставляет функционал для:
- Создания и управления событиями (онлайн/оффлайн форматы, даты регистрации и проведения).
- Регистрации пользователей на события.
- Управления организациями и их участниками (роли администратора/участника).
- Интеграции с LLM (Google Gemini) для обработки форм (модуль `forms-wrapper`).

## Технологии

- **Python**: 3.12+
- **Django**: 6.0+
- **Django Ninja**: для API
- **Poetry**: управление зависимостями
- **Google GenAI (Gemini)**: ИИ-функционал

## Установка и запуск

### Предварительные требования

Убедитесь, что у вас установлены:
- Python 3.12 или выше
- Poetry (инструмент управления зависимостями)

### 1. Клонирование репозитория

```bash
git clone git@github.com:SlavaKemDev/spi-platform.git
cd spi-platform
```

### 2. Установка зависимостей

Используйте Poetry для установки всех необходимых пакетов:

```bash
poetry install
```

### 2.1. Установка браузеров (модуль forms_wrapper)

Модуль `forms_wrapper` использует Playwright для работы с динамическими веб-страницами при автоматическом анализе сторонних форм регистрации. Poetry устанавливает только Python-пакет, браузеры нужно скачать отдельно:

```bash
playwright install chromium firefox
```

На Linux (сервере) также нужны системные зависимости:

```bash
sudo playwright install-deps chromium firefox
```

### 3. Настройка окружения

Необходимо настроить переменные окружения. Создайте файл `.env` в корне проекта.

**Важно:** Секретный ключ Django (`SECRET_KEY`) теперь хранится в `.env` и должен быть установлен для запуска проекта.

Пример содержимого `.env`:
```dotenv
SECRET_KEY="ваш_сгенерированный_секретный_ключ"
GEMINI_API_KEY="ваш_ключ_api"
```

Сгенерировать новый `SECRET_KEY` можно с помощью команды:
```bash
python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
```

### 4. Применение миграций

Перед запуском сервера примените миграции базы данных:

```bash
poetry run python manage.py migrate
```

### 5. Создание суперпользователя (опционально)

Для доступа к админ-панели создайте администратора:

```bash
poetry run python manage.py createsuperuser
```

### 6. Запуск сервера разработки

Запустите локальный сервер Django:

```bash
poetry run python manage.py runserver
```

После запуска сервер будет доступен по адресу: http://127.0.0.1:8000/

## Структура проекта

- `core/`: Основные настройки проекта.
- `events/`: Приложение для работы с событиями и регистрациями.
- `organizations/`: Приложение для управления организациями.
- `users/`: Приложение для управления пользователями.
- `forms-wrapper/`: Модуль для работы с формами и интеграции с LLM.
